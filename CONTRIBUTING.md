# Contributing to NetPulse UpGuardX

Thank you for your interest in contributing to NetPulse UpGuardX! We welcome all contributions, from bug fixes and feature requests to documentation improvements.

## Code of Conduct

Please be respectful and professional in all interactions within this project.

## How to Contribute

1.  **Fork the Repository:** Create a personal fork of the project on GitHub.
2.  **Create a Branch:** Create a new branch for your changes: `git checkout -b feature/my-feature` or `git checkout -b fix/my-fix`.
3.  **Make Changes:** Implement your changes, following the existing code style and adding tests if applicable.
4.  **Commit Changes:** Commit your changes with clear and descriptive messages: `git commit -m "feat: adds my new feature"`.
5.  **Push to Your Fork:** Push your branch to your personal fork.
6.  **Create a Pull Request:** Open a pull request against the `main` branch of the official repository.

## Development Environment Setup

Please refer to the [Developer Guide](docs/developer/DEVELOPER.md) for detailed instructions on setting up your development environment.

## Reporting Issues

If you find a bug or have a feature request, please search the existing issues before opening a new one. If the issue is a security vulnerability, please refer to the [Security Policy](SECURITY.md) instead of opening a public issue.
